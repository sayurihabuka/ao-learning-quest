# 語彙力UP1300 作問共通テンプレート

このテンプレートは、`ao-learning-quest` の「問題作成｜語彙力UP1300」スレッドに貼って使うための共通指示です。  
作問時は、安定版ルール `docs/vocab1300_question_rules.md` を前提にします。

---

## 使い方

`pXXX` の部分を対象ページ番号に置き換えて使います。

例：

```txt
p015を作問してください。
```

---

## 共通指示テンプレート

```txt
pXXXを作問してください。

最新版の `vocab1300_question_rules.md` を前提にしてください。

作業内容：
1. pXXXから語句・読み・意味を抽出してください
2. 各語句に適した問題タイプを判定してください
3. `extractedWords` を作成してください
4. 問題をJSON形式で作成してください
5. `answerIndex` は元データ上の正解位置として設定してください
6. アプリ側で選択肢シャッフルする前提で、極端な偏りだけチェックしてください
7. `answerIndexCheck` を出してください
8. 最後に、作問ルールへの修正候補があれば `ruleDraftRevisionNotes` にまとめてください

出力形式：
{
  "sourcePage": XXX,
  "pageTypeJudgement": {},
  "extractedWords": [],
  "questions": [],
  "answerIndexCheck": {},
  "ruleDraftRevisionNotes": []
}

注意：
・教材本文や教材画像はそのまま使わないでください
・青ちゃん向けのオリジナル例文・オリジナル問題にしてください
・小学3年生が読んで理解しやすい文にしてください
・問題文は短くしてください
・選択肢は4択にしてください
・選択肢には、青ちゃんが選びそうな誤答も混ぜてください
・完全に不自然な誤答ばかりにしないでください
・解説は短く、何を覚えればよいかが分かる形にしてください
・画像が必要な問題は `image` に仮の画像パスを入れてください
・画像は将来的に生成AIで作るオリジナル画像を使う前提にしてください
・画像には文字・数字・記号を入れない前提にしてください
・`answerIndex` は 0〜3 の数値にしてください
・作問データ上の `answerIndex` は完全均等でなくてよいですが、極端な偏りは避けてください
```

## 問題タイプ判定の目安

### meaning_context

語句の意味を、短い文脈から選ぶ問題。

例：
- したたか
- 口下手
- 客観的
- 陰口

基本形：

```json
{
  "type": "meaning_context",
  "word": "口下手",
  "reading": "くちべた"
}
```

---

### reading_context

同じ表記でも、文脈によって読み方と意味が変わる語。

例：
- 生物：せいぶつ／なまもの
- 人事：ひとごと／じんじ
- 見物：けんぶつ／みもの

基本形：

```json
{
  "type": "reading_context",
  "word": "生物",
  "answerReading": "なまもの"
}
```

解説は、次の形を基本にする。

```txt
この文では〜という意味なので、読み方は『〜』です。
```

---

### age_word

年齢を表す特別な語。

例：
- 不惑
- 還暦
- 喜寿
- 卒寿
- 白寿

基本形：

```json
{
  "type": "age_word",
  "word": "還暦",
  "reading": "かんれき"
}
```

解説には、読み方と何歳を表すかを入れる。

---

### image_word

絵を見て、場面や語句を選ぶ問題。

例：
- 傘かしげ
- 肩ひき
- こぶし腰浮かせ

p014のような気づかい・マナー・しぐさを表す語は、次の形にする。

```json
{
  "type": "image_word",
  "subtype": "manner_word",
  "word": "傘かしげ",
  "image": "images/manners_kasakashige.png",
  "hint": "雨の日、すれちがう人の傘にぶつからないようにします。"
}
```

---

### month_hint

月の異名を見て、何月を表すかを選ぶ問題。

例：
- 睦月
- 弥生
- 文月
- 葉月
- 霜月
- 師走

基本形：

```json
{
  "type": "month_hint",
  "subtype": "month_word",
  "word": "弥生",
  "reading": "やよい",
  "month": 3
}
```

解説には、読み方と何月を表すかを入れる。  
由来は断定しすぎず、「〜と言われています」と書く。

---

### counter_image

物を見て、数え方を選ぶ問題。

例：
- とうふ：一丁
- たんす：一棹
- そば：一人前
- 布：一反
- うさぎ：一羽
- くつ：一足

基本形：

```json
{
  "type": "counter_image",
  "word": "一足",
  "reading": "いっそく",
  "counter": "足",
  "counterReading": "そく",
  "counterTarget": "くつ",
  "image": "images/counter_kutsu.png"
}
```

## まず3問だけ試作したい場合

ページ全体をいきなり作る前に、タイプ確認をしたい場合は、こちらを使います。

```txt
pXXXでテスト作問してください。

最新版の `vocab1300_question_rules.md` を前提にしてください。

作業内容：
1. pXXXから語句・読み・意味を抽出してください
2. 問題タイプを判定してください
3. まず3問だけ試作してください
4. JSON形式で出力してください
5. `answerIndex` は元データ上の正解位置として設定してください
6. アプリ側で選択肢シャッフルする前提で、極端な偏りだけチェックしてください
7. 最後に `answerIndexCheck` と、作問ルールへの修正候補を `ruleDraftRevisionNotes` にまとめてください

注意：
・教材本文や教材画像はそのまま使わないでください
・青ちゃん向けのオリジナル問題にしてください
・問題文は短くしてください
・画像が必要な問題は `image` に仮の画像パスを入れてください
・新しいtypeやsubtypeが必要そうな場合は、最後に提案してください
```

## 残り問題を作る場合

3問テスト後、残りを作るときはこちらを使います。

```txt
pXXXの残り問題も作成してください。

前回作成した問題と合わせて、pXXX全問として確認したいです。

作業内容：
1. 残りの語句・読み・意味を確認してください
2. 前回の問題と同じ形式で問題を作成してください
3. type / subtype は前回の方針に合わせてください
4. JSON形式で出力してください
5. `answerIndex` は元データ上の正解位置として設定してください
6. アプリ側で選択肢シャッフルする前提で、極端な偏りだけチェックしてください
7. pXXX全問としての `answerIndexCheck` を出してください
8. 最後に、作問ルールへの修正候補を `ruleDraftRevisionNotes` にまとめてください

注意：
・教材本文や教材画像はそのまま使わないでください
・青ちゃん向けのオリジナル問題にしてください
・前回のanswerIndexに偏りがある場合は、可能であれば残り問題で分散してください
・ただし、選択肢の自然さを壊してまで均等にする必要はありません
```

## 出力チェック

作問後に確認すること。

```txt
・JSONとして壊れていないか
・sourcePage が正しいか
・id が vocab1300_pXXX_001 の形式になっているか
・type が適切か
・subtype が必要な問題に入っているか
・choices が4択になっているか
・answerIndex が 0〜3 の数値になっているか
・answerIndex が正解選択肢を指しているか
・image が必要な問題に仮パスが入っているか
・教材本文や教材画像をそのまま使っていないか
・青ちゃん向けに文が難しすぎないか
・解説が短く分かりやすいか
・answerIndexCheck が出ているか
・ruleDraftRevisionNotes が出ているか
```
