# vocab1300 画像生成ルール

**対象**: 青のまなびクエスト — 語彙力UP1300  
**作成日**: 2026-07-02  
**更新日**: 2026-07-04

---

## 0. 生成運用の基本方針

### 生成ツール

画像生成は **Google AI Studio を主軸** とする。

| 用途 | ツール |
|---|---|
| 本番生成の主軸 | Google AI Studio |
| Google AI Studio で品質が出ない場合の補助 | GP画像生成（ChatGPT） |

**基本フロー:**

```
Google AI Studio で生成
  ↓
GP が画像を確認
  ↓
問題があれば Google AI Studio で再生成
  ↓
再生成しても改善しない場合は GP画像生成へ切り替える
```

### 要注意カテゴリ

以下のカテゴリは Google AI Studio での生成が難しい場合がある。試作結果を見て GP生成への切り替えを検討する。

**weather_word（気象・風系）:** 霧 / 靄 / 霞 / おろし / 山背 / 木枯らし など  
**day_time_word（時間帯）:** たそがれ / あけぼの / 未明 / 宵の口 / 夜半 など

### 1枚ずつ生成する

- 画像は **1枚ずつ** 生成する（まとめて生成しない）
- Google AI Studio はファイル名を指定して保存できないため、**保存後に正式ファイル名へリネーム** する
- 生成画像は **GPが確認** し、以下のいずれかを判断する
  - 採用
  - プロンプトを修正して Google AI Studio で再生成
  - Google AI Studio での再生成を繰り返しても改善しないため、GP画像生成へ切り替える

**GPの確認観点:**

- 語句の意味に合っているか
- 小さく表示しても意味が分かるか
- 共通ルールや subtype 別ルールに違反していないか
- 類似語と区別できるか
- 枠・文字・不要な矢印・過剰な描き込みがないか

### 採用後の作業

採用と判断した画像は、必要に応じて以下を行う。

1. PNG形式であることを確認（JPEG等の場合は `sips` でPNG変換）
2. `pngquant` 等で圧縮（画質劣化がなければ実施）
3. 正式ファイル名にリネーム
4. `generated-images/trial/` 等に保管
5. 最終的に `app/images/` へ配置

## 1. 共通ルール

すべての画像に適用する基本方針。

| 項目 | 内容 |
|---|---|
| 出典 | 教材画像は使わない。生成AIで作るオリジナル画像のみ使用する |
| 背景 | **白背景**。余計な背景要素は入れない |
| スタイル | 白黒のシンプルな手描き風線画 |
| 対象年齢 | 小学生向け教材イラスト |
| サイズ・比率 | **正方形** |
| 文字・記号 | **文字・数字・記号・吹き出し・ラベルは絶対に入れない** |
| 枠・フレーム | **四角い枠・フレームは絶対に入れない** |
| 黒ベタ | **黒いベタ塗りを多くしすぎない** |
| ハッチング | **細かいハッチングを増やしすぎない** |
| 描き込み | **描き込みすぎない。主役が小さくならないようにする** |
| 構図 | 主役を大きく描く。余計な背景を入れない |
| 視認性 | **小さく表示しても何が描かれているか分かる絵にする** |

## 1.5 部分名称画像の矢印ルール（例外）

体・衣服・建物・道具の **一部分を問う画像** では、必要に応じて矢印を1本だけ入れてよい。

**対象 subtype:**

- `clothing_word`（例: 袂 → 有効だった）
- `body_part_word`（例: 土ふまず → 有効だった）
- `house_part_word`（例: 鴨居 → 有効だった）
- `old_life_tool_word`（例: いろり → 有効だった）
- `cooking_word`（必要な場合のみ）

**矢印のルール:**

| 項目 | 内容 |
|---|---|
| 本数 | **1本だけ** |
| 文字ラベル | **入れない**（「ここ」などの文字も不可） |
| 丸囲み | 原則使わない |
| スタイル | 黒のシンプルな線 |
| 指す対象 | 問いたい部分のみ |
| 全体感 | 説明図になりすぎないようにする |

**矢印を使わない subtype:**

`weather_word` / `day_time_word` / `manner_word` では矢印は使わない。自然現象や動作は、矢印ではなく絵の構図で表現する。

## 2. subtype別の描き方方針

### counter_image（数え方・助数詞）

- 数え方（助数詞）に対応するモノを主役に描く
- 「1つ」「1本」のように、数量が分かる描き方にする
- モノの形・特徴が一目で分かるシルエットにする
- 同じ助数詞でも対象が異なる語がある（例: 一丁 → そろばん / 豆腐）ため、モノ自体を明確に描く
- 人物は原則不要。モノだけで成立する構図にする

### manner_word（礼儀・作法・しぐさ）

- 人が礼儀・作法・しぐさをしているシーンを描く
- 動作の「何をやっているか」が分かるシンプルな人物イラスト
- 人物は2人以上のことが多い（相手がいる場面）
- 動作の方向・関係性が伝わる構図にする
- 表情ではなく動作・ポーズで内容を表現する

### clothing_word（和服・衣服の部位）

- 和服・衣服を着た人物を描き、注目部位を分かりやすく示す
- 全体と注目部位の両方が伝わる構図にする
- 部位の形・位置が視覚的に理解できるようにする
- 似た語（留め袖・振り袖など）は、袖の長さ・全体の雰囲気など違いが分かるように描く
- 裾・袂・懐など部位名の場合は、対象部分を大きく・はっきり示す構図にする
- 矢印ルールは §1.5 を参照

### weather_word（天気・気象・季節の風）

**注意: 30問あり、似た語が多い。混同しないよう特に区別を意識する。**

#### 雨・水分系（p38 / p50）

雨の強さ・降り方・季節感が区別できるようにする。

- 篠突く雨: 縦に細い線が密集して激しく降っている。**枠・フレームが入りやすいため「四角い枠は絶対に入れない」を徹底する**
- にわか雨: 急な雨、晴れ間との対比で表現
- 夕立: 夏、激しい雨と暗い空
- 通り雨: ざっと降ってすぐやむ、軽い雨
- 菜種梅雨: 春、花（菜の花）と細かい雨
- 空梅雨: 梅雨なのに晴れた空（雨なし）
- しぐれ: 秋冬の端境期、軽くぱらぱら
- 五月雨: 梅雨のしとしと長雨
- 小糠雨: 霧のような極細の雨
- 狐の嫁入り: 晴れているのに雨（太陽と雨の同時描写）
- 春雨: 春のしとしと雨
- 入梅: 梅雨の始まり（初めて傘を差すシーンなど）

#### 風系（p62 / p74）

風の向き・強さ・季節感が区別できるようにする。

- はやて: 突然の強風、人が驚いている
- 野分け: 秋冬の荒々しい風、草木が大きくなびく
- 春一番: 暖かい南風、春らしさ
- 山背: 東北の海側に吹く冷たい風（山と海のシルエット）
- おろし: 山から下へ吹き下ろす風。**矢印や流線は使わず、木・草・服・髪が山の上から下へなびく様子だけで風向きを表す**
- 木枯らし: 秋冬の冷たい強風、枯れ葉が舞う
- つむじ風: うずまき状の風
- 空っ風: 冷たく乾いた北風、雲なし
- 花風: 桜の花びらを揺らす風
- 追い風: 人の進行方向と同じ向きの風
- 青嵐: 初夏の若葉を揺らす強い風
- 向かい風: 人の進行方向と逆向きの風

#### 雲・霧・霞系（p86）

雲・霧・霞・靄は、空や見え方の違いが分かるようにする。

- 入道雲: 夏、もくもくと高く盛り上がった大きな積乱雲
- 霞: 春、遠くがぼんやり見える（遠景がぼやけた景色）
- 霧: 地面近くが白くなり、視界が悪い（近距離の白いもや）。**白背景に埋もれないよう、家・木・道の輪郭を残しつつ、地面近くのもやを少し濃くする**
- ひつじ雲: 小さな丸い雲が群れで並んでいる
- 靄: 空気全体がぼんやり（霧より薄いもや）
- いわし雲: 魚のうろこ状の小さな雲が広がっている

### day_time_word（一日の時間帯）

空・地平線・建物などで時間帯を表現する。明るさ・太陽や月の位置・星の有無で区別する。

- あけぼの: 夜明け前後、空の端がほんのり明るい
- 未明: 夜中を過ぎたが、まだ暗い
- 昼下がり: 明るい昼の空、太陽が少し西側
- たそがれ: 夕方、日が沈みかける直前。**黒いベタ塗りや細かいハッチングを増やしすぎない。地平線の近くに少し明るさを残し、「日が沈みかける直前」と分かる構図にする**
- 宵の口: 夕暮れ直後、日が沈んだばかり
- 夜半: 深夜、暗く静かな夜

1枚でも明暗の違いが分かる構図にする（空のグラデーション、星・月・太陽の有無）。

### body_part_word（体の部位）

- 体のシルエット全体の中で、対象部位を大きく・明確に示す
- §1.5 の矢印ルールに従い、必要に応じてラベルなしの黒い矢印を1本だけ使って対象部位を指してよい
- 位置が分かりにくい部位（きびす、土ふまず）は足のシルエットの中で該当箇所を際立たせる
- 正面・側面・背面など、対象部位が最も分かりやすいアングルを選ぶ

### house_part_word（日本家屋の部位・建具）

- 日本家屋の外観または室内を描き、注目部分が分かるようにする
- 似た語は区別を明確にする
  - 鴨居（上の溝のある横木）と敷居（下の溝のある横木）は上下の位置が分かる構図
  - 軒（屋根の張り出し）とひさし（玄関・窓上の小屋根）は外観で区別
  - ふすま（布張りの引き戸）と障子（白い紙張りの引き戸）は素材感・模様で区別
- 建具（障子・ふすま）は、開閉している様子より「どんな見た目か」を中心に描く
- 矢印ルールは §1.5 を参照

### cooking_word（料理の動作）

- 調理動作を主役にする
- 食材・調理器具・手の動きで「何をしているか」を表現する
- 似た動作を区別する
  - あえる（野菜と調味料を和える）・からめる（ソースをまとわせる）・まぶす（表面に粉をつける）は動作とターゲットの違いで区別
  - 湯がく（さっと熱湯に通す）・ふかす（蒸気で蒸す）・煮る（水と一緒に火にかける）は調理方法で区別
- 動作が完了した状態ではなく、動作の最中を描く

### old_life_tool_word（昔の生活道具）

- 道具の形と使い方の両方が伝わるイラストにする
- 現代の子どもが見て「何に使うものか」が想像できる描き方にする
  - 火鉢: 炭を入れて、手を温めたりする昔の道具
  - おひつ: 炊いたご飯を入れておく、木でできた丸い容器
  - 縁台: 縁側や軒先に置く、板でできた腰かけ台
  - すだれ: 細い竹や葦を並べてつないだ、窓などに掛ける日よけ
  - 蚊帳: 蚊を防ぐために寝るところに吊る、薄い布の網
  - いろり: 部屋の床に四角く切り込んで作った、火を起こす炉
- 道具単体だけでなく、使われている場面（人が使っているシーン）を入れてもよい
- 矢印ルールは §1.5 を参照

## 4. 画像生成フロー

1. `image-prompts/vocab1300_image_targets_v1.json` で画像対象・問題ID・正式ファイル名を確認する
2. Google AI Studio で1枚生成する
3. 生成画像をそのまま保存する
4. 問題ID・語句・元ファイル名・正式ファイル名を記録する（後述の対応記録ルールを参照）
5. GP が画像をチェックする
6. 採用 / AI Studio で再生成 / GP画像生成へ切り替え を判断する
7. 採用画像を PNG形式でなければ `sips` でPNG変換する
8. 正式ファイル名（§3 のルールに従う）へリネームする
9. 必要に応じて `pngquant` で圧縮する（画質劣化がなければ実施）
10. `generated-images/` 等に元画像・採用画像を保管する
11. `app/images/` へ配置する
12. 問題JSONの `image` パスと照合する
13. PNG形式・解像度・縦横比・破損・画質を最終確認する

### 運用メモ

- Google AI Studio は保存ファイル名が自動になるため、**生成後のリネーム作業が前提**
- 採用画像は `generated-images/` 等に保管後、最終的に `app/images/` に配置する
- 圧縮には `pngquant` を使う（インストール済み）
- 圧縮前後のファイルサイズと削減率を報告する
- 圧縮後に PNG形式・解像度・縦横比・画質を目視確認してから元ファイルと置き換える

### 生成画像の対応記録

Google AI Studio から画像を保存した直後に、次の4項目を記録する。

- 問題ID
- 語句
- AI Studio から保存された元ファイル名
- JSON で指定されている正式ファイル名

記録例:

    問題ID    : vocab1300_p086_003
    語句      : 霧
    元ファイル名: Generated Image ... .png
    正式ファイル名: weather_kiri.png

**複数画像をまとめて保存してから記憶を頼りに対応付けない。**  
保存するたびに対応を記録し、採用確定後にクロが正式ファイル名へリネームする。
