## 2. ツールの目的

`ao-learning-quest` は、青ちゃんが暗記系の学習内容を忘却曲線に沿って復習できるWeb学習ツールです。

v1では、最初の教材として **語彙力UP1300** を扱います。

目的は、単に1回正解することではなく、日をまたいだ復習によって語句を定着させることです。

---

## 3. v1の対象教材

v1の対象教材は以下です。

- 語彙力UP1300

教材スキャンから語句・読み・意味・問題形式を確認しますが、アプリに入れる問題は教材本文をそのまま使わず、青ちゃん向けのオリジナル問題として作り直します。

---

## 4. 将来的な拡張方針

v1では語彙力UP1300を対象にしますが、将来的には以下にも拡張できる設計にします。

- 算数の単位・公式・暗記
- 漢字
- ことわざ・慣用句
- 理科
- 社会
- 歴史人物
- 英単語

そのため、問題データには `source` や `type` を持たせ、教材や問題形式を後から増やせる構造にします。

---

## 5. 学習目標

v1の目標は、語彙力UP1300の1300語のうち、**90%を「定着○」にすること**です。

---

## 6. 定着○の条件

「定着○」は、1回正解しただけでは認定しません。

定着○の条件は以下です。

```txt
日をまたいで2回以上○
かつ
level 2以上
```

選択問題は偶然正解する可能性があるため、日をまたいだ正解を重視します。

---

## 7. 1日の出題数

1日の出題数は、基本 **40問** とします。

---

## 8. 1周目の考え方

1周目は「完全暗記」ではなく、1300語を以下のように仕分けることを目的にします。

- すでに覚えている語
- 少しあやしい語
- まだ覚えていない語

1周目から復習も混ぜ、△や×の語を翌日以降に再出題します。

---

## 9. 1周目の出題ルール

基本は以下です。

```txt
新規30問 + 前日△×10問
```

ただし、前日の△×数に応じて自動調整します。

| 前日の△×数 | 今日の出題 |
|---:|---|
| 0〜9語 | 新規35問＋復習5問 |
| 10〜19語 | 新規30問＋復習10問 |
| 20語以上 | 新規20問＋復習20問 |

---

## 10. 判定ルール

○と×はアプリ側で自動判定します。  
△のみ、青ちゃんが自己申告します。

| 結果 | 判定方法 | 処理 |
|---|---|---|
| ○ | 正解＋迷わない | level +1 |
| △ | 正解後に「ちょっとあやしい」ボタンを押す | levelそのまま、翌日もう一度 |
| × | 不正解 | level -1、翌日もう一度 |

※ levelは0より下げません。

---

## 11. △ボタン仕様

△は、正解したあとにだけ選べるようにします。

### ボタン名

```txt
ちょっとあやしい
```

### 表示する説明

青ちゃんが吹き出しでつぶやいているような形で、以下の内容を表示します。

```txt
こんな時は押してね
・迷った
・たまたま正解した
・消去法で当たった
・意味が少し不安
```

### △の処理

△ボタンを押した場合は、以下の処理にします。

- levelは上げない
- `unsureCount` を +1
- `nextReviewAt` は翌日
- 定着○にはしない

---

## 12. 忘却曲線ルール

復習間隔は以下の通りです。

| level | 状態 | 次回復習 |
|---:|---|---:|
| 0 | 未定着 | 翌日 |
| 1 | ○1回 | 3日後 |
| 2 | 日をまたいで○が増えてきた | 7日後 |
| 3 | かなり覚えた | 14日後 |
| 4 | 定着候補 | 30日後 |
| 5 | ほぼ定着 | 60日後 |

### ○のとき

- `level` を +1
- `correctCount` を +1
- `streak` を +1
- `lastAnsweredAt` を更新
- `nextReviewAt` を level に応じて設定
- 定着条件を満たしたら `isMastered: true`

### △のとき

- `level` はそのまま
- `unsureCount` を +1
- `nextReviewAt` は翌日
- 定着○にはしない

### ×のとき

- `level` を -1
- `wrongCount` を +1
- `streak` を 0
- `nextReviewAt` は翌日
- 定着○にはしない

## 13. 問題形式

語彙力UP1300では、v1時点で以下の5パターンを想定します。

| type | 内容 | 問題形式 | イラスト |
|---|---|---|---|
| `meaning_context` | 例文中の語句の意味 | 短い文脈＋意味4択 | 不要 |
| `reading_context` | 読み方で意味が変わる語 | 例文＋読み方4択 | 不要 |
| `image_word` | 絵を見て場面・語句を選ぶ | イラスト＋言葉4択 | 必要 |
| `month_hint` | 月の異名 | ヒント文＋何月か4択 | 任意 |
| `counter_image` | 物の数え方 | イラスト＋数え方4択 | 必要 |

---

## 14. 問題作成ルール

問題作成はGPが担当します。

### 共通ルール

- 教材本文をそのまま使わない
- 青ちゃん向けのオリジナル例文に作り直す
- 小学3年生が読める長さにする
- 文脈から意味が推測できるようにする
- 選択肢は4つ
- 正解は1つだけ
- 正解位置はア・イ・ウ・エに分散させる
- 解説は短く、わかりやすくする
- 誤答選択肢は意味が重なりすぎないようにする
- ひっかけすぎない
- 迷わせることより、語句の理解と定着を優先する

---

## 15. 正解位置ルール

問題作成時は、正解が毎回同じ位置にならないようにします。

- 正解が毎回アにならないようにする
- 同じページ内でア・イ・ウ・エに分散させる
- JSONでは `answerIndex` で管理する
- アプリ側で表示時に選択肢をシャッフルする設計も検討可

## 17. Firebase進捗データ仕様

Firebase Realtime Databaseには、問題本文ではなく、青ちゃんの進捗だけ保存します。

### 進捗データ例

```json
{
  "level": 2,
  "mark": "circle",
  "correctCount": 3,
  "wrongCount": 1,
  "unsureCount": 2,
  "streak": 1,
  "firstAnsweredAt": 1760000000000,
  "lastAnsweredAt": 1760000000000,
  "nextReviewAt": 1760259200000,
  "stableCorrectCount": 2,
  "isMastered": true
}
```

### mark

| mark | 意味 |
|---|---|
| `circle` | ○ |
| `triangle` | △ |
| `cross` | × |

### isMastered

`isMastered` は、定着○の条件を満たしたら `true` にします。

条件は以下です。

```txt
日をまたいで2回以上○
かつ
level 2以上
```

## 31. 同日やり直し機能

### 目的

通常40問で△・×になった問題を、その日のうちに最大2周まで再出題します。

目的は長期定着の判定ではなく、以下の通りです。

```txt
- 正しい答えをその日のうちに確認する
- 間違えたまま終わらない
- 満足して学習を終えられるようにする
```

「今日のクリア ≠ 定着○」であることが重要な考え方です。

---

### 1日の学習フロー

```txt
① 通常40問
②（△・×が1問以上あれば）同日やり直し・1周目
③（1周目でまだ不正解があれば）同日やり直し・2周目
④ 最後の振り返り画面
```

---

### ① 通常40問

既存のルール（セクション7〜10）通りに出題・判定します。

この判定結果だけが以下に影響します。

```txt
- level
- 定着○回数
- 次回復習日
```

---

### ② 同日やり直し・1周目

通常40問で△・×だったすべての問題を再出題します。

1周目では、以下を変更しません。

```txt
- 問題文
- 例文
- 選択肢
- 並び順
- 正解位置
```

1周目で正解した問題は、画面上で「今日のクリア」状態にします。  
ただし、内部処理では以下を変更しません。

```txt
- level
- 定着○回数
- nextReviewAt（△・×で設定された翌日復習をそのまま維持）
```

---

### ③ 同日やり直し・2周目

1周目でまだ不正解だった問題だけを再出題します。

問題文・例文・選択肢・並び順・正解位置は1周目と同じです。

2周目で正解した問題は、画面上で「2周目でクリア」状態にします。  
内部処理ではlevel・定着○回数・nextReviewAtを変更しません。

2周目でも不正解だった問題は「明日もう一度」とし、その日はそれ以上繰り返しません。

#### 2周目でも不正解だった場合の翌日方針

同日やり直し結果による再判定はしません。

- 通常40問の初回△・×判定で設定された `nextReviewAt` をそのまま維持する
- `level` を追加で下げない
- 定着○回数を変更しない
- 定着判定には影響させない
- 最後の表示は「明日もう一度」
- 翌日は通常の復習問題として出題する

---

### ④ 最後の振り返り画面

2周目終了後、通常40問で△・×だったすべての問題を一覧表示します。

表示項目：

```txt
- 語句
- 読み
- 正しい答え
- 簡単な解説
- 今日の結果（今日のクリア／2周目でクリア／明日もう一度）
```

通常40問で○だった問題は振り返り一覧に表示しません。

---

### 回答判定と内部処理

| 判定 | levelへの影響 | 定着○回数 | nextReviewAt |
|---|---|---|---|
| 通常40問：○ | +1 | 対象 | levelに応じて設定 |
| 通常40問：△ | 変更なし | 対象外 | 翌日 |
| 通常40問：× | -1（0未満にしない） | 対象外 | 翌日 |
| 同日やり直しで正解 | 変更なし | 対象外 | 変更なし |
| 同日やり直しで不正解（2周目含む） | 変更なし | 対象外 | 変更なし |

---

### 翌日復習との違い

| 項目 | 同日やり直し | 翌日の復習 |
|---|---|---|
| 問題文 | 同じ | 同じ |
| 例文 | 同じ | 同じ |
| 選択肢 | 同じ | シャッフルする |
| 同日の正解 | levelに反映しない | levelに反映する |

---

### 1日の問題数の扱い

基本出題数は40問のままです。

同日やり直しは追加練習として扱います。

```txt
通常問題：40問
同日やり直し：△・×数に応じて追加
```

実際の回答数は40問を超える場合があります。

---

### 進捗記録の候補フィールド

実装時に参考にするための候補です。Firebaseスキーマはこの時点では確定していません。

```txt
initialResult          … 通常40問での結果
sameDayRetryRound1     … 同日やり直し1周目の結果
sameDayRetryRound2     … 同日やり直し2周目の結果
sameDayFinalStatus     … 同日やり直しの最終状態
```

`sameDayFinalStatus` の候補値：

```txt
cleared_round1    … 1周目で正解
cleared_round2    … 2周目で正解
```
